## 1.1 EventTarget
```idl
[Exposed=*]
interface EventTarget {
    // 事件监听管理
    undefined addEventListener(DOMString type, EventListener? callback, optional (AddEventListenerOptions or boolean) options = {});
    undefined removeEventListener(DOMString type, EventListener? callback, optional (EventListenerOptions or boolean) options = {});
    boolean dispatchEvent(Event event);
};
```
| 成员 | 类型/签名 | 功能描述 |
|------|----------|---------|
| `addEventListener` | `undefined` | 在目标上注册指定类型的事件监听器。`options` 可配置 `capture`（捕获阶段）、`once`（只触发一次）、`passive`（不调用 `preventDefault()`）、`signal`（AbortSignal 取消监听） |
| `removeEventListener` | `undefined` | 移除已注册的事件监听器。必须提供与注册时相同的 `type`、`callback` 和 `capture` 标志 |
| `dispatchEvent` | `boolean` | 手动派发事件。返回 `false` 表示事件被取消了（`preventDefault()` 被调用）。事件按捕获→目标→冒泡顺序传播 |

## 1.2 Node
```idl

interface Node : EventTarget {
    // 节点类型常量

    const unsigned short ATTRIBUTE_NODE = 2; // 遗留

```
| 成员 | 类型 | 功能描述 |
|------|------|---------|
| `nodeType` | `unsigned short` (只读) | 返回节点类型常量（如 `ELEMENT_NODE = 1`） |
| `nodeName` | `DOMString` (只读) | 元素返回大写标签名；Document 返回 `"#document"`；Text 返回 `"#text"` 等 |
| `baseURI` | `USVString` (只读) | 节点的基础 URL，用于解析相对 URL。受 `<base>` 标签影响 |
| `isConnected` | `boolean` (只读) | 节点是否连接到文档树（即是否在 `Document` 的 descendant 链中） |
| `ownerDocument` | `Document?` (只读) | 节点所属的 `Document`。`Document` 自身和 `DocumentType` 的 `ownerDocument` 为 `null` |
| `parentNode` | `Node?` (只读) | 父节点。根节点、已移除节点返回 `null` |
| `parentElement` | `Element?` (只读) | 父元素节点。若父节点不是元素（如 `DocumentFragment`）返回 `null` |
| `childNodes` | `NodeList` (只读) | 包含所有子节点的实时集合（live collection） |
| `firstChild` / `lastChild` | `Node?` (只读) | 第一个/最后一个子节点 |
| `previousSibling` / `nextSibling` | `Node?` (只读) | 前一个/后一个兄弟节点 |
| `nodeValue` | `DOMString?` | 对 `Text`/`Comment`/`ProcessingInstruction` 可读写文本内容；对 `Element`/`Document` 等返回 `null` |
| `textContent` | `DOMString?` | 获取/设置节点及其后代的文本内容。设置时会移除所有子节点并替换为单个 `Text` 节点 |
| `cloneNode(deep)` | `Node` | 复制节点。`deep = true` 时深拷贝（含所有后代）；`false` 时仅复制节点自身。不复制事件监听器 |
| `hasChildNodes()` | `boolean` | 是否有子节点 |
| `insertBefore(node, child)` | `Node` | 在 `child` 之前插入 `node`。`child` 为 `null` 时等同于 `appendChild` |
| `appendChild(node)` | `Node` | 将 `node` 追加到子节点列表末尾。若 `node` 已在树中，先移除再插入 |
| `replaceChild(node, child)` | `Node` | 用 `node` 替换 `child`，返回被移除的 `child` |
| `removeChild(child)` | `Node` | 从子节点列表移除 `child` |
| `normalize()` | `undefined` | 合并相邻的 `Text` 节点，移除空的 `Text` 节点 |
| `replaceChildren(...nodes)` | `undefined` | 移除所有现有子节点，插入新节点（可混用 `Node` 和 `DOMString`，字符串自动转为 `Text`） |
| `prepend(...nodes)` / `append(...nodes)` | `undefined` | 在子节点列表开头/末尾插入 |
| `contains(other)` | `boolean` | 判断 `other` 是否是当前节点的后代（含自身） |
| `compareDocumentPosition(other)` | `unsigned short` | 返回两个节点在文档中的相对位置位掩码（`DOCUMENT_POSITION_DISCONNECTED`, `PRECEDING`, `FOLLOWING`, `CONTAINS`, `CONTAINED_BY`） |
| `isEqualNode(other)` | `boolean` | 深度比较两个节点结构、类型、名称、属性、内容是否完全相同 |
| `getRootNode({composed})` | `Node` | 返回根节点。`composed = true` 时穿透 Shadow DOM 边界返回 `Document` |

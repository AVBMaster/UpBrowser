HTML Elements Complete Reference for Browser Engine Development
Based on the HTML Living Standard document you provided, here is a comprehensive summary of all HTML elements with their specifications, organized for AI reference when developing a browser rendering engine.

1. Document Structure Elements
html
Categories: None

Context: Document root element

Content model: head element followed by body element

Tag omission: Start tag can be omitted if first inside is not a comment; end tag can be omitted if not immediately followed by comment

Interface: HTMLHtmlElement extends HTMLElement

Special behavior: Root of HTML document; encouraged to specify lang attribute

head
Categories: None

Context: First element in html

Content model: Metadata content elements (zero or more); exactly one title unless title from higher-level protocol

Tag omission: Start tag can be omitted if empty or first child is element; end tag can be omitted if not followed by whitespace/comment

Interface: HTMLHeadElement extends HTMLElement

title
Categories: Metadata content

Context: In head with no other title elements

Content model: Text (not inter-element whitespace)

Tag omission: Neither tag omissible

Interface: HTMLTitleElement extends HTLElement

Properties: text IDL attribute

base
Categories: Metadata content

Context: In head with no other base elements

Content model: Nothing (void)

Tag omission: No end tag

Attributes: href (document base URL), target (default navigable)

Interface: HTMLBaseElement extends HTMLElement

Special: First base with href sets frozen base URL; only first with target matters

link
Categories: Metadata content, flow content, phrasing content (if allowed in body)

Context: Where metadata expected; in noscript child of head; where phrasing expected (if allowed in body)

Content model: Nothing (void)

Tag omission: No end tag

Attributes: href, crossorigin, rel, media, integrity, hreflang, type, referrerpolicy, sizes, imagesrcset, imagesizes, as, blocking, color, disabled, fetchpriority

Interface: HTMLLinkElement extends HTMLElement (includes LinkStyle)

Special: External resource links and hyperlinks; processing model based on rel keywords

meta
Categories: Metadata content, flow content (if itemprop present), phrasing content (if itemprop present)

Context: In head with charset or http-equiv; where metadata expected

Content model: Nothing (void)

Tag omission: No end tag

Attributes: name, http-equiv, content, charset, media

Interface: HTMLMetaElement extends HTMLElement

Standard names: application-name, author, description, generator, keywords, referrer, theme-color, color-scheme

style
Categories: Metadata content

Context: Where metadata expected; in noscript child of head

Content model: Text (conformant style sheet)

Tag omission: Neither tag omissible

Attributes: media, blocking, title

Interface: HTMLStyleElement extends HTMLElement (includes LinkStyle)

Properties: disabled IDL attribute

2. Sectioning Elements
body
Categories: None

Context: Second element in html

Content model: Flow content

Tag omission: Start tag can be omitted if empty or first not whitespace/comment (except meta/noscript/link/script/style/template); end tag can be omitted if not followed by comment

Event handlers: onafterprint, onbeforeprint, onbeforeunload, onhashchange, onlanguagechange, onmessage, onmessageerror, onoffline, ononline, onpageswap, onpagehide, onpagereveal, onpageshow, onpopstate, onrejectionhandled, onstorage, onunhandledrejection, onunload

Interface: HTMLBodyElement extends HTMLElement (includes WindowEventHandlers)

article

Content model: Flow content

Tag omission: Neither tag omissible

Interface: HTMLElement

Semantics: Self-contained composition (forum post, article, blog entry, comment)

section
Categories: Flow content, sectioning content, palpable content

Context: Where sectioning content expected

Content model: Flow content

Tag omission: Neither tag omissible

Semantics: Generic section, thematic grouping with heading

nav

Content model: Flow content

Tag omission: Neither tag omissible

Interface: HTMLElement

Semantics: Section with navigation links

aside

Context: Where sectioning content expected

Content model: Flow content

Tag omission: Neither tag omissible

Semantics: Tangentially related content (sidebars, pull quotes)

h1-h6
Categories: Flow content, heading content, palpable content

Content model: Phrasing content

Tag omission: Neither tag omissible

Interface: HTMLHeadingElement extends HTMLElement

Semantics: Headings with level (1-6)

hgroup
Categories: Flow content, heading content, palpable content

Context: Where heading content expected

Content model: Zero or more p elements, then one heading (h1-h6), then zero or more p elements

Tag omission: Neither tag omissible

Interface: HTMLElement

Semantics: Heading with subheading/tagline

header
Categories: Flow content, palpable content

Context: Where flow content expected

Tag omission: Neither tag omissible

footer
Categories: Flow content, palpable content

Context: Where flow content expected

Content model: Flow content (no header/footer descendants)

address
Categories: Flow content, palpable content

Context: Where flow content expected

Content model: Flow content (no heading/sectioning/header/footer/address descendants)

Tag omission: Neither tag omissible

Interface: HTMLElement

Semantics: Contact information for nearest article or body

dl
Categories: Flow content, palpable content (if has name-value group)

Content model: Zero or more groups of dt+dd; or one or more div elements

Tag omission: Neither tag omissible

Semantics: Description list (name-value groups)

dt
Categories: None

Context: Before dd/dt inside dl or div children of dl

Content model: Flow content (no header/footer/sectioning/heading descendants)

Tag omission: End tag can be omitted if followed by dt or dd

Interface: HTMLElement

Categories: None

Context: After dt/dd inside dl or div children of dl

Content model: Flow content

Tag omission: End tag can be omitted if followed by dd or dt or no more content

Interface: HTMLElement

figure
Categories: Flow content, palpable content

Context: Where flow content expected

Content model: Either figcaption followed by flow, or flow followed by figcaption, or just flow

Tag omission: Neither tag omissible

Interface: HTMLElement

Semantics: Self-contained content with optional caption

figcaption
Categories: None

Context: First or last child of figure

Content model: Flow content

Tag omission: Neither tag omissible

Interface: HTMLElement

main
Categories: Flow content, palpable content

Context: Where flow content expected (must be hierarchically correct)

Content model: Flow content

Tag omission: Neither tag omissible

Interface: HTMLElement

Semantics: Dominant contents of document

Constraint: Only one main without hidden attribute; ancestors limited to html, body, div, form without accessible name, autonomous custom elements

search
Categories: Flow content, palpable content

Context: Where flow content expected

Content model: Flow content

Tag omission: Neither tag omissible

Interface: HTMLElement

Semantics: Part of document with search/filter functionality

div
Categories: Flow content, palpable content, select/optgroup/option inner content

Context: Where flow content expected; as child of dl; as descendant of option/optgroup/select

Content model: Depends on context (flow or specific content)

Tag omission: Neither tag omissible

Interface: HTMLDivElement extends HTMLElement

Semantics: Generic container

10. Scripting Elements
script
Categories: Metadata content, flow content, phrasing content, script-supporting

Context: Where metadata/phrasing/script-supporting expected

Content model: Depends on src and type

Tag omission: Neither tag omissible

Attributes: type, src, nomodule, async, defer, blocking, crossorigin, referrerpolicy, integrity, fetchpriority

type values: empty/JS MIME → classic script; "module" → module script; "importmap" → import map; "speculationrules" → speculation rules; others → data block

Interface: HTMLScriptElement extends HTMLElement

Properties: text

Static method: supports(type)

noscript
Categories: Metadata content, flow content, phrasing content, select/optgroup inner content

    value: free-form string

    value: free-form string

type_states:
  hidden:
    description: Value not intended to be examined by user
    constraints: Barred from constraint validation
    special: If name="_charset_", value attribute must be omitted
    attributes_applicable: [autocomplete, dirname]
    idl_mode: default
    events: none
    
  text:
    description: One line plain text edit control
    value_sanitization: Strip newlines
    constraints: No LF/CR characters allowed
    attributes_applicable: [autocomplete, dirname, list, maxlength, minlength, pattern, placeholder, readonly, required, size]
    idl_mode: value
    events: [input, change]
    
  search:
    description: Search control (styling differs from text)
    value_sanitization: Strip newlines
    attributes_applicable: [autocomplete, dirname, list, maxlength, minlength, pattern, placeholder, readonly, required, size]
    idl_mode: value
    events: [input, change]
    
  tel:
    description: Telephone number input
    value_sanitization: Strip newlines
    attributes_applicable: [autocomplete, dirname, list, maxlength, minlength, pattern, placeholder, readonly, required, size]
    idl_mode: value
    events: [input, change]
    
  url:
    description: Absolute URL input
    value_sanitization: Strip newlines, then strip leading/trailing whitespace
    constraints: Suffers from type mismatch if not empty and not valid absolute URL
    attributes_applicable: [autocomplete, dirname, list, maxlength, minlength, pattern, placeholder, readonly, required, size]
    idl_mode: value
    events: [input, change]
    
  email:
    description: Email address(es) input
    multiple_mode:
      - Without multiple: single email address
      - With multiple: comma-separated email address list
    value_sanitization:
      single: Strip newlines, strip leading/trailing whitespace
      multiple: Split on commas, trim each, rejoin with commas
    constraints: Type mismatch if not valid email address(es)
    attributes_applicable: [autocomplete, dirname, list, maxlength, minlength, multiple, pattern, placeholder, readonly, required, size]
    idl_mode: value
    events: [input, change]

    description: Password input (obscured)
    value_sanitization: Strip newlines
    attributes_applicable: [autocomplete, dirname, maxlength, minlength, pattern, placeholder, readonly, required, size]
    idl_mode: value
    events: [input, change]

    description: Date picker (YYYY-MM-DD)
    value_sanitization: If not valid date string, set to empty
    step_scale: 86,400,000 (days to milliseconds)
    default_step: 1 day
    attributes_applicable: [autocomplete, list, max, min, readonly, required, step]
    idl_mode: value (with valueAsDate, valueAsNumber)
    events: [input, change]
    conversion:
      string_to_number: milliseconds from 1970-01-01 to parsed date
      number_to_string: valid date string from milliseconds

    description: Month picker (YYYY-MM)
    value_sanitization: If not valid month string, set to empty
    step_scale: 1
    default_step: 1 month
    attributes_applicable: [autocomplete, list, max, min, readonly, required, step]
    idl_mode: value (with valueAsDate, valueAsNumber)
    events: [input, change]

    description: Radio button (group selection)

    value_mode: default/on
    attributes_applicable: [checked, required]
    idl_properties: [checked, value]
    events: [input, change]

  file:
    description: File upload control
    value_mode: filename

    attributes_applicable: [accept, multiple, required]

  submit:
    description: Submit button
    value_mode: default
    attributes_applicable: [dirname, formaction, formenctype, formmethod, formnovalidate, formtarget, popovertarget, popovertargetaction]

    description: Reset button
    value_mode: default
    attributes_applicable: [popovertarget, popovertargetaction]
    idl_mode: default

  button:
    description: Button with no default behavior
    value_mode: default
    attributes_applicable: [popovertarget, popovertargetaction]
    idl_mode: default

    applies: [date, month, week, time, datetime-local, number, range]

    getter: Returns datalist element referenced by list attribute

    effect: Shows picker UI if supported

    applies: [text, search, tel, url, password]

    applies: [text, search, tel, url, password]

Key areas for rendering engine implementation:
